# Simplicity Rules

Prioritize:

- visual quality
- readability
- maintainability

Keep Angular logic minimal.

Avoid:

- unnecessary abstractions
- overengineered architecture
- excessive services
- complicated state management
- advanced watchers
- premature optimization

Use:

- static mock data
- simple arrays/constants
- minimal component logic

Do not simulate backend behavior.

Prefer:

- simpler code
- cleaner templates
- better spacing
- stronger visual polish
